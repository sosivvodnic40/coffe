## Слайд 3 — Цель и задачи

**Цель:** создать веб-сайт с серверной частью для ресторана «Cappuccino».

**Задачи:**
1. Анализ предметной области и аналогов

6. Тестирование и документирование

## Слайд 6 — Архитектура системы

```
Браузер → React SPA → REST API (Express) → SQLite
```

- Клиент-серверная архитектура
- SPA + REST API
- Разделение на публичную и административную части
- См. `diploma/03_Schemes/architecture.md`

## Слайд 7 — База данных

6 таблиц: locations, menu_categories, menu_items, reservations, promotions, admin_users

- Связь locations → reservations (1:N)
- Связь menu_categories → menu_items (1:N)
- Индексы по дате/времени и статусу брони

## Слайд 8 — Функции для гостя

- Информация о ресторане и контакты
- Меню по категориям (цены в ₸)
- Акции и спецпредложения
- Галерея интерьера
- **Онлайн-бронирование** с кодом подтверждения CP-XXXXXX
- Проверка статуса брони по коду

---

## Слайд 9 — Панель администратора

- URL: `/admin`
- Авторизация по JWT (email + пароль)
- Статистика: всего броней, ожидающих, гостей сегодня
- Список броней с фильтрацией по статусу
- Смена статуса: pending → confirmed / cancelled / completed

---

## Слайд 10 — REST API

Публичные: `/menu`, `/locations`, `/promotions`, `/reservations`  
Админ: `/admin/login`, `/admin/stats`, `/admin/reservations`

- 11 endpoints
- Валидация Zod, коды ошибок 400/401/404/409
- Спецификация: `diploma/05_DB_API/api-spec.md`

---

## Слайд 11 — Безопасность

- Хеширование пароля администратора (bcrypt)
- JWT-токен для админ-запросов
- Rate limit на бронирование (10 POST/час)
- Helmet + CORS
- Параметризованные SQL-запросы
